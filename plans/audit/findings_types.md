# Types / Schema Contract Findings

## Summary
The Python hook deliberately preserves malformed session entries on disk (entries missing `state`/`lastActivity`/`startedAt`), but the Rust `SessionInfo` struct marks those exact fields as *required* (no `#[serde(default)]`). Because `StatusData` deserializes as one `HashMap<String, SessionInfo>`, a single such preserved entry makes `serde_json::from_str::<StatusData>` abort the **whole** parse — dropping every session and eventually tripping the self-repair that wipes `sessions.json`. The numeric/token typing (i64 tokens, f64 timestamps, `pid: u32`) and the JSONL `Value`-based parser are otherwise resilient.

## Findings

### F-types-001: Missing `#[serde(default)]` on required SessionInfo fields makes one malformed entry abort the whole sessions.json parse
- **Severity:** critical
- **Confidence:** 95
- **Files:** `cue-desktop/src-tauri/src/models.rs:14-20` ; `cue-desktop/src-tauri/src/models.rs:80-83` (`StatusData`) ; `cue-desktop/src-tauri/src/session_monitor.rs:159` ; `hooks/cue-hook:138-168` (`_validate_sessions`) ; `tests/hooks/test_cue_hook.py:61-65`
- **What:** `SessionInfo.id`, `.workspace`, `.state`, `.last_activity` (JSON `lastActivity`), and `.started_at` (JSON `startedAt`) have **no** `#[serde(default)]` (only `state_changed_at`/`source`/`inputTokens`/`outputTokens`/`model`/`activeSubagents` and the optionals do). On the writer side, the hook's `_validate_sessions` is explicitly documented and unit-tested (`test_cue_hook.py:61-65` keeps entry `"abc": {"id":"abc","workspace":"/Users/x/y"}` — no `state`, no `lastActivity`, no `startedAt`) to *preserve* such entries rather than drop them, and writes the full `status["sessions"]` map back to disk verbatim in both the main path (`hooks/cue-hook:978`, `:993`) and the quick-write path (`:309`, `:319`). When Rust reads that file at `session_monitor.rs:159` via `serde_json::from_str::<StatusData>(&content)`, serde requires every non-`default` field of every map value; a single value missing `state`/`lastActivity`/`startedAt` returns `Err`, so the entire `StatusData` (all sessions) fails to deserialize — not just the one bad entry. There is no per-entry resilience (no container `default`, no custom deserializer, no `filter_map`).
- **Why it matters:** One malformed entry — from version skew, a manual edit, an interrupted/older hook, or any future field rename — silently blanks the *entire* dashboard. Worse, the parse-failure arm (`session_monitor.rs:166-222`) counts consecutive failures and after 5 polls (~5s) **renames the file aside and seeds `{"sessions":{}}`** (`:199-208`), destroying every live session's record on disk. The writer's "never drop a live session" guarantee is fully defeated by the reader's strict contract.
- **Suggested fix:** Make the reader as tolerant as the writer. Minimum change: add `#[serde(default)]` to `id`, `workspace`, `state`, `last_activity`, `started_at` in `SessionInfo` — the admission filter at `session_monitor.rs:276-280` already drops empty-id / unsanitizable-workspace entries, so a defaulted-empty bad entry gets filtered instead of aborting everything. Alternative: deserialize `sessions` as `HashMap<String, serde_json::Value>` (or `Option<SessionInfo>`) and skip per-entry failures, preserving the good ones.
- **Verification:** Rust test asserting per-entry resilience: `serde_json::from_str::<StatusData>(r#"{"sessions":{"abc":{"id":"abc","workspace":"/x"},"def":{"id":"def","workspace":"/y","state":"working","lastActivity":1.0,"startedAt":1.0}}}"#)` returns `Err` today (missing field `state`); after the fix it yields 1 admissible session. Writer side: `python3 -m pytest tests/hooks/test_cue_hook.py -k validate -q` (lines 61-65 confirm `"abc"` is preserved on disk).

## Out of scope
- **`pid: Option<u32>` (models.rs:56) vs hook `os.getppid()` (cue-hook:287,937).** A PID > `u32::MAX` would abort the parse, but `pid` is `#[serde(default)]` (missing is fine) and real PIDs never approach u32 max. Negligible. (Caveat: a present-but-wrong-type value still aborts the whole map per F-types-001, but the trigger is implausible.)
- **`active_subagents`/`inputTokens`/`outputTokens` as i64 (models.rs:33,36,42).** i64 covers any realistic token count; per-field defaults mean missing → 0, not abort. No summation overflow in `jsonl_parser.rs:895-898`. Fine.
- **`jsonl_parser.rs` token/usage typing (425-442, 884-902).** Untyped `Value` + `.get(f).and_then(as_i64).unwrap_or(0)` per field — fully resilient; missing/renamed/wrong-type usage field → 0, never aborts. Safe.
- **State string matching.** No catch-all corrupts a known state: `models.rs:444-469` passes unknowns through (`other => other` name; `_ => "✓"` is only the icon glyph), and `session_monitor.rs` uses explicit `matches!` allowlists that exclude unknown states. `"ended"` comes via the hook's `remove` path (`cue-hook:804`), not `action`; `"clearing"` is never written by the hook (`cue-hook:602`; `session_monitor.rs:1123-1131`). No mismatch.
- **No `deny_unknown_fields` anywhere** in the target files — forward-compat preserved; unknown fields ignored.
- **`SandboxSessionPayload` (lib.rs:579-589)** is Rust↔Rust (Tauri command), not Python↔Rust; defaults `active_subagents`/`source` as `Option` and omits `stateChangedAt`/`pid` (harmless — both `#[serde(default)]` on read).
- **Incoming permission HTTP payload (lib.rs:1700-1724)** parsed as untyped `Value` accepting snake_case+camelCase with `.unwrap_or("unknown")`. `PermissionLogEntry` (permission_log.rs:88) read with `filter_map(...ok())`, per-line resilient and Rust-written. Robust.
- **`env_detect.rs`** only `Serialize`s `EnvironmentInfo`; never deserializes `sessions.json`. Outside the contract.
- **TypeScript mirror drift** — out of scope per brief.
